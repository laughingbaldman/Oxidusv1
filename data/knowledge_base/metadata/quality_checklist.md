# Knowledge Base Quality Checklist

- [ ] File names follow the naming standard
- [ ] Topic type is assigned and recorded
- [ ] Source URL and license captured in source_registry.json
- [ ] Source trust tier meets or exceeds curation minimum
- [ ] Provenance log entry recorded for each write
- [ ] Summary exists in summaries/ when content is curated
- [ ] Cross-cutting tags added when relevant
- [ ] Duplicate detection performed for new ingests
- [ ] Archive path set for superseded files
- [ ] Indexes updated after changes
